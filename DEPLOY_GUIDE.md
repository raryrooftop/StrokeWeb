# 🚀 배포 지침 (Deployment Guide)

이 문서는 AI(Antigravity)가 이 프로젝트를 배포할 때 반드시 준수해야 하는 절차를 담고 있습니다. 임의의 프로젝트에 잘못 배포되는 것을 방지하기 위한 필수 지침입니다.

---

## 1. 배포 전 필수 확인 절차

AI는 배포 명령(`gcloud app deploy` 등)을 내리기 전에 반드시 다음 단계를 수행해야 합니다.

1.  **프로젝트 목록 확인**: `gcloud projects list` 명령어를 실행하여 현재 접근 가능한 GCP 프로젝트 목록을 확인합니다.
2.  **대상 프로젝트 매칭**: 
    - `credentials.json` 또는 환경 설정 파일에 적힌 프로젝트 ID를 확인합니다.
    - 현재 로컬에 설정된 프로젝트(`gcloud config get-value project`)가 실제 배포 대상이 맞는지 대조합니다.
3.  **사용자 최종 승인**: 
    - 확인된 프로젝트 ID를 사용자에게 제시하며, **"해당 프로젝트([ID])로 배포를 진행할까요?"**라고 한국어로 질문하고 확답을 받습니다.

## 2. 배포 실행 규칙

사용자의 승인을 받은 후에는 다음 명령어를 순차적으로 실행합니다.

```bash
# 1. 배포 대상 프로젝트 강제 고정
gcloud config set project [확인된-프로젝트-ID]

# 2. 배포 실행 (사용자 확인을 거쳤으므로 --quiet 옵션 사용 가능)
gcloud app deploy --quiet
```

## 3. 배포 완료 후 보고

배포가 완료되면 다음 정보를 사용자에게 요약하여 보고합니다.
- 배포된 프로젝트 ID
- 서비스 접속 URL
- 주요 수정 사항 요약

---

> **주의**: 여러 병원/부서 프로젝트를 관리하는 환경이므로, 프로젝트 ID가 헷갈리지 않도록 매번 배포 시마다 목록을 다시 확인해야 합니다.
